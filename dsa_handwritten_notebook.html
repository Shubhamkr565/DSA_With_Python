1. Sliding Window Array / String

★When to use: subarray / substring + some condition on elements
→Keywords: contiguous, max/min subarray, longest substring
Fixed Window — window size = k given
1.Build first window (i = 0 to k-1)
2.Slide: add arr[right], remove arr[left], advance both
3.Track best at each step
Variable Window — shrink when condition breaks
1.Expand right freely
2.Shrink left until valid again
3.Update answer on every valid window
left = 0 for right in range(n): window.add(arr[right]) # expand while window_invalid(): window.remove(arr[left]); left++ # shrink ans = max(ans, right - left + 1) # update
⚠Mistake: incrementing left more than once per step in variable window!
⏱Time: O(n)   Space: O(1) or O(k)

2. Two Pointer Array

★When to use: sorted array + pair / triplet condition, palindrome
→Keywords: two sum sorted, 3sum, remove dups, container water
Opposite Ends l=0, r=n-1 → move inward sum < target → l++ sum > target → r--
Same Direction slow writes, fast reads fast skips, slow places used in: remove dups, merge
l, r = 0, n-1 while l < r: s = arr[l] + arr[r] if s == target: return [l, r] elif s < target: l++ # need bigger else: r-- # need smaller
⏱Time: O(n)   Space: O(1)

3. Binary Search Sorted / Monotonic

★When to use: sorted data OR "if X works then X+1 also works"
→Keywords: find min/max, first/last position, feasibility check
Classic → find target in sorted array On Answer → "find minimum X such that f(X) = True"
lo, hi = 0, n-1 while lo < hi: mid = lo + (hi - lo) // 2 ← no overflow! if feasible(mid): hi = mid # search left else: lo = mid+1 # search right return lo # answer!
⚠NEVER write (lo+hi)//2 — causes integer overflow in C++!
⚠Decide boundary: hi = mid vs hi = mid-1 based on if mid is a valid candidate
⏱Time: O(log n)   Space: O(1)
init =
{0:1}!

4. Prefix Sum + HashMap Array

★When to use: subarray sum = k, range queries, frequency
→Keywords: sum equals k, count subarrays, range sum
# Prefix sum build pre[0] = 0 for i in range(n): pre[i+1] = pre[i] + arr[i] sum(i, j) = pre[j+1] - pre[i] ← O(1) query!
# Subarray sum = k (HashMap trick) seen = {0: 1} ← CRITICAL: empty subarray total = count = 0 for x in arr: total += x count += seen.get(total - k, 0) seen[total] = seen.get(total, 0) + 1
⚠Always initialize seen = {0: 1} or you miss subarrays starting at index 0
⏱Time: O(n)   Space: O(n)

6. Monotonic Stack Stack

★When to use: next/prev greater / smaller element problems
→Keywords: daily temps, stock span, largest histogram, trap water
Increasing Stack Pop when cur < top → Next Smaller Element
Decreasing Stack Pop when cur > top → Next Greater Element
# Next Greater Element st, res = [], [-1] * n for i, v in enumerate(arr): while st and arr[st[-1]] < v: res[st.pop()] = v # found NGE! st.append(i) # push INDEX not value
⏱Time: O(n) — each element pushed+popped once. Space: O(n)

7. Trees: DFS + BFS Tree

BFS (Level Order) Use deque queue Shortest path (unweighted) Process entire level at once
DFS (Recursion) Pre / In / Post order Path sum, subtree, height Base case = None → 0
# BFS level order q = deque([root]) while q: for _ in range(len(q)): # whole level node = q.popleft() if node.left: q.append(node.left) if node.right: q.append(node.right)
# DFS height / depth def dfs(node): if not node: return 0 return 1 + max(dfs(node.left), dfs(node.right))

9. Dynamic Programming DP

★When to use: optimal substructure + overlapping subproblems
→Keywords: max/min ways, count paths, knapsack, subsequence, partition
5-Step DP Framework
1.Define — dp[i] = "max/min/count of ... up to index i"
2.Base case — dp[0] = ? (smallest input)
3.Recurrence — dp[i] = f(dp[i-1], dp[i-2], ...)
4.Order — fill left to right (usually)
5.Answer — dp[n] or max/min of dp[]
DP Types:
→1D: dp[i] — house robber, climbing stairs, LIS
→2D: dp[i][j] — LCS, edit distance, grid paths
→Knapsack: dp[i][w] — 0/1, unbounded, subset sum
→Interval: dp[l][r] — matrix chain, burst balloons
# LCS (Longest Common Subsequence) dp = [[0]*(m+1) for _ in range(n+1)] for i in range(1, n+1): for j in range(1, m+1): if s1[i-1] == s2[j-1]: dp[i][j] = dp[i-1][j-1] + 1 else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])

10. Heap + Greedy Heap

★Heap: when you need repeated min/max efficiently
★Greedy: when local optimal = global optimal (must prove!)
Min Heap (default) heapq.heappush(h, val) heapq.heappop(h) → Top-K smallest
Max Heap (Python) heappush(h, -val) -heappop(h) → Top-K largest
# K largest using min-heap of size k import heapq; heap = [] for x in nums: heapq.heappush(heap, x) if len(heap) > k: heapq.heappop(heap) return heap[0] # kth largest!
⏱Heap ops: O(log n)   Top-K: O(n log k)   Build heap: O(n)

11. Backtracking Recursion

★When to use: find ALL solutions, permutations, combinations, sudoku
→Keywords: all subsets, all paths, N-Queens, word search, generate
Universal Template
def backtrack(path, choices): if is_solution(path): # base case result.append(path.copy()); return for choice in choices: if is_valid(choice): path.append(choice) # MAKE choice backtrack(path, next_choices) path.pop() # UNDO choice ←KEY
⚠The undo step is everything. Without it, state bleeds between branches!
★3 questions: What is my choice? What is my base case? What do I undo?
⏱Time: O(n! or 2ⁿ) — expensive but necessary for exhaustive search
